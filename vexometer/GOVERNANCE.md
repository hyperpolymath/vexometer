# Governance

## Project Lead

**Jonathan D.A. Jewell** - Benevolent Dictator for Now (BDfN)

## Decision Making

- Minor changes: Any maintainer
- Major changes: Project lead approval
- Breaking changes: Community discussion + project lead

## Becoming a Maintainer

1. Consistent quality contributions
2. Demonstrated understanding of project goals
3. Invitation from existing maintainers

## Meetings

No regular meetings. Async communication via:
- GitLab issues
- Merge request discussions

## Amendments

This governance model may be updated via merge request.
